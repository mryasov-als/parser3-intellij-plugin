# Parser 3 Support

Плагин для IntelliJ IDEA с поддержкой языка Parser3.

Parser3 - серверный язык с шаблонной моделью, методами, классами, `@USE`/`^use[]`, встроенными типами и активным смешением Parser3-кода с HTML, CSS, JavaScript и SQL. Цель плагина - сделать работу с такими проектами ближе к привычной IDE-разработке: подсветка, автодополнение, навигация, документация, форматирование и безопасный рефакторинг путей.

## Демо

[Видео с возможностями плагина](https://github.com/mryasov-als/parser3-intellij-plugin/releases/download/v0.7.39/parser3plugin_demo.mp4)

## Возможности

### Поддержка файлов Parser3

- Распознавание файлов `.p` и `.p3`.
- Лексер и парсер для Parser3-конструкций.
- Подсветка директив, методов, классов, переменных, строк, комментариев, операторов и встроенных сущностей.
- Настраиваемая цветовая схема `Editor | Color Scheme | Parser 3`.
- Комментирование Parser3-кода через стандартные действия IDE.
- Сопоставление парных скобок `[]`, `()`, `{}`.
- Сворачивание блоков кода.

### Автодополнение

- Методы Parser3: `^method[]`, `^self.method[]`, `^MAIN:method[]`, `^BASE:method[]`.
- Статические вызовы и конструкторы классов: `^Class:method[]`, `^Class::create[]`.
- Пользовательские классы, `@BASE`, partial-классы и наследование.
- Встроенные классы Parser3 и их методы.
- Переменные: `$var`, `$self.var`, `$MAIN:var`, `$BASE:var`.
- Свойства объектов через `$object.property` и `@GET_property`.
- Ключи hash и колонки table, если структура выводится из кода.
- Параметры итераторов `foreach`, `for`, `select`, `sort`.
- Пути в `@USE` и `^use[]`.
- Опции `@OPTIONS`.
- Пользовательские шаблоны в `Settings | Editor | Parser 3 | Templates`.
- Контекстные аргументы и значения в `Settings | Editor | Parser 3 | Contextual argument completion`.

### Навигация и поиск

- Переход к объявлениям методов, классов и переменных.
- Навигация по `@USE` и `^use[]` к подключаемым файлам.
- Навигация по `@BASE` к базовому классу.
- Переход от вызовов объектных методов к методам класса переменной, если тип удалось вывести.
- Поиск использований для Parser3-файлов и подключений.
- Учёт `auto.p`, `@autouse`, `CLASS_PATH`, `document_root` и позиционной видимости.
- Popup выбора, если найдено несколько подходящих объявлений.

### Анализ типов и структуры данных

- Индексация методов, классов, `@USE`/`^use[]`, вызовов методов, переменных и CSS-классов.
- Вывод типа переменной из конструкторов пользовательских и встроенных классов.
- Вывод типа результата метода по документационным комментариям и `$result`.
- Поддержка table-колонок из `^table::create[]`, `^table::sql[]`, `^file:list[]` и похожих конструкций.
- Поддержка hash-ключей, read-chain inference и вложенных цепочек.
- Учёт локальности переменных через `@OPTIONS locals`, `[locals]` и явные списки локальных переменных.

### Встроенная документация

- Quick Documentation для встроенных классов и методов Parser3.
- Подсказки по сигнатурам и параметрам.
- Ссылки на bundled-документацию Parser3.
- Документация для пользовательских методов из комментариев.

### HTML, CSS, JavaScript и SQL внутри Parser3

- Инъекция HTML в Parser3-файлах.
- Инъекция CSS внутри `<style>`.
- Инъекция JavaScript внутри `<script>`.
- Инъекция SQL в Parser3 SQL-блоках.
- Настраиваемые SQL-инъекции в `Settings | Editor | Parser 3 | SQL injections`.
- Навигация и completion CSS-классов из `class="..."` к определениям в `<style>` внутри Parser3-файлов.

### Форматирование и редактирование

- Форматирование Parser3-кода через стандартный formatter IDE.
- Code Style settings для Parser3.
- Автоотступы при Enter между скобками.
- Обработка Tab и Shift+Tab с учётом Parser3-комментариев.
- Автоформатирование закрывающих скобок.
- Корректировка Backspace и Paste для Parser3-контекстов.

### Рефакторинг путей

- Обновление путей в `@USE` и `^use[]` при перемещении или переименовании файлов и директорий.
- Учёт относительных путей, абсолютных путей от `document_root`, fallback-логики Parser3 и `CLASS_PATH`.
- Предупреждения о проблемных ссылках через уведомления IDE.

### Настройки

Раздел настроек находится в:

`Settings | Editor | Parser 3`

Доступны:

- основные настройки Parser3-проекта;
- пользовательские шаблоны;
- contextual argument completion;
- SQL injections.

## Требования

- IntelliJ IDEA 2026.1 или новее.
- Java 21 для сборки проекта.
